1. Método Constructor
Explicacion:
El constructor es un método que se ejecuta cuando se crea un objeto de una clase. Sirve para darle valores iniciales a los atributos del objeto cuando se crea.

Ubicacion:
Alimento.java - línea 20
Perecedero.java - línea 10
Refrigerado.java - línea 14

2. Métodos de acceso (Getters y Setters)
Explicacion:
Son métodos que permiten obtener o cambiar el valor de los atributos de una clase. Se usan porque los atributos suelen ser privados y no se pueden modificar directamente desde otras clases.

Ubicacion:
Alimento.java - línea 34 y línea 80
Perecedero.java - línea 30 y línea 40

3. Método toString()
Explicacion:
Este método sirve para mostrar la información de un objeto en forma de texto. Se usa cuando queremos imprimir los datos del objeto de forma más clara.

Ubicacion:
Alimento.java - línea 106
Perecedero.java - línea 50
Refrigerado.java - línea 60

4. Modificador static
Explicacion:
La palabra static indica que ese atributo o método pertenece a la clase y no a un objeto en específico. Por eso todos los objetos comparten ese mismo valor.

Ubicacion:
Alimento.java - línea 14
Refrigerado.java - línea 7 y línea 40

5. Modificador final
Explicacion:
Final se usa para indicar que algo no se puede cambiar. Puede aplicarse a variables, métodos o clases.

Ubicacion:
Alimento.java - atributo MARCA_CERTIFICADA
Alimento.java - método obtenerInfo()
certificado.java - línea 1

6. Visibilidades
Explicacion:
Las visibilidades sirven para controlar desde dónde se pueden usar los atributos o métodos de una clase. Por ejemplo, private significa que solo se puede usar dentro de la misma clase y public significa que se puede usar desde cualquier clase.

Ubicacion:
Alimento.java - línea 6
Alimento.java - línea 34
Refrigerado.java - línea 7

7. Multiplicidades
Explicacion:
Las multiplicidades indican que una clase puede tener varios objetos de otra clase. En el código se representa usando arreglos.

8. Sobreescritura de métodos
Explicacion:
La sobreescritura ocurre cuando una clase hija usa un método de la clase padre pero le cambia su funcionamiento.

9. Sobrecarga de métodos
Explicacion:
La sobrecarga ocurre cuando hay varios métodos con el mismo nombre pero con diferentes parámetros.

Ubicacion:
Alimento.java - línea 20

10. Creación de objetos
Explicacion:
Es cuando se crea un objeto a partir de una clase usando la palabra new para poder usar sus atributos y métodos.

11. Agregaciones
Explicacion:
La agregación es cuando una clase contiene objetos de otra clase como parte de ella, pero esos objetos pueden existir también por separado.

12. Herencia
Explicacion:
La herencia permite que una clase pueda usar atributos y métodos de otra clase. Esto ayuda a reutilizar código.
